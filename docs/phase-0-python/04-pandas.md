# Урок 04 — pandas: DataFrame, selection, groupby, EDA

## Що це

**pandas** — бібліотека для роботи з табличними даними. Два ключові типи:

- **`Series`** — одна колонка з ім'ям і індексом рядків. Ментально — це `numpy.ndarray` плюс дві речі: назва колонки і мітки для кожного елемента.
- **`DataFrame`** — двовимірна таблиця з іменованими колонками. Ментально — словник Series-ів, склеєних по спільному індексу рядків.

Під капотом і Series, і DataFrame зберігають числові дані у **`np.ndarray`**. Тому вся швидкість NumPy лишається — broadcasting, векторна арифметика, dtype. pandas додає **шар семантики** зверху: імена колонок, індекс рядків, зручне читання CSV, фільтри, групування.

Просто: **pandas — це NumPy з ярликами**.

## Для чого

У реальному ML ти майже ніколи не працюєш з голими матрицями. Ти працюєш з таблицями, де:
- Колонки означають різні речі (`age`, `salary`, `city`) з різними dtype.
- Є пропуски (`NaN`), які треба явно обробити.
- Треба фільтрувати за умовами: "тільки клієнти старше 30", "тільки з Києва".
- Треба групувати і агрегувати: "середня зарплата по містах".
- Треба зчитувати з CSV/parquet/бази і зберігати назад.

Голий `ndarray` цього не вміє — там немає імен колонок, немає dtype-mix, `NaN` тільки для float. **pandas — це NumPy + SQL + Excel в одному API.**

Перший день з новим датасетом — це майже завжди 4-5 pandas-викликів. Це і називається **EDA** (Exploratory Data Analysis).

## Розбір

### 1. Створення DataFrame

Найпростіший спосіб — словник, де ключ = ім'я колонки, значення = список даних:

```python
import pandas as pd

people = pd.DataFrame({
    "name":   ["Ігор", "Петро", "Юля", "Катя", "Леся"],
    "age":    [25, 23, 22, 22, 20],
    "salary": [2000.0, 1800.0, 1800.0, 1900.0, 1500.0],
    "city":   ["Київ", "Львів", "Одеса", "Київ", "Львів"],
})
```

**Ментальна модель:**
- Ключі словника → назви колонок.
- Списки → колонки, кожен елемент = один рядок.
- **Всі списки мають бути однакової довжини** — це і буде кількість рядків.

pandas автоматично додає **індекс** для рядків — за замовчуванням `0..n-1`. Це ліва колонка яку ти бачиш у виводі.

### 2. Первинний огляд датасету — три обов'язкові кроки

Це те, що робиться на будь-якому новому CSV в першу секунду:

```python
people.head()      # перші 5 рядків
people.info()      # структура: dtype колонок, non-null count, memory
people.describe()  # count/mean/std/min/25%/50%/75%/max для числових
```

**Порядок і логіка:**
- **`.head()`** — окомір: як воно взагалі виглядає, скільки колонок, які значення.
- **`.info()`** — паспорт таблиці: скільки рядків, які типи, скільки non-null у кожній колонці. Одразу видно де пропуски.
- **`.describe()`** — числовий підсумок: чи є дивні max/min, який розкид, чи схоже на реальність.

За замовчуванням `.describe()` показує **тільки числові колонки**. Над рядками нема сенсу рахувати `.mean()`. Щоб побачити стати для рядків:

```python
people.describe(include="str")   # тільки рядкові — count, unique, top, freq
people.describe(include="all")   # все разом, з NaN де стать не застосовується
```

Для рядків показує **інший набір статистик**:
- `count` — скільки non-null.
- `unique` — скільки різних значень.
- `top` — найчастіше значення (mode). Якщо всі унікальні — просто перше.
- `freq` — скільки разів зустрічається `top`.

**Ключове:** `NaN` у виводі `include="all"` — це **"не застосовується"**, а не "невідомо". `mean` для колонки імен — безглуздо, pandas ставить `NaN`. У pandas `NaN` перевантажений, це нюанс.

### 3. Selection — три ортогональні способи

Це місце, де pandas найбільше плутає новачків. Три способи, і **кожен потрібен для свого**.

#### А) `df["col"]` — квадратні дужки з іменем

```python
people["salary"]              # Series (одна колонка)
people[["name", "city"]]      # DataFrame (список імен → таблиця)
```

Одна назва → Series. **Список** назв → DataFrame. Різні дужки (`[...]` vs `[[...]]`) — не помилка, це семантика:
- Внутрішні дужки — це Python-список назв колонок.
- Зовнішні — це синтаксис `df[...]` для індексації.

**Gotcha:** якщо забудеш подвійні дужки і напишеш `people["name", "city"]` — Python розпарсить це як **tuple** `("name", "city")`, pandas сприйме як "одна колонка з іменем-кортежем" і кине `KeyError`.

**Ще gotcha:** `people["salary"]` — pandas-idiomatic. Не використовуй `.get("salary")` (dict-style API): при опечатці колонки вернеться `None` без помилки → біль при дебагу. `[]` дає `KeyError` — fail-fast, одразу видно.

#### Б) `.loc[row_label, col_label]` — доступ по мітках

```python
people.loc[1, "name"]                 # значення в рядку з міткою 1, колонці "name"
people.loc[:, "salary"]               # ціла колонка "salary" (як Series)
people.loc[0:2, ["name", "city"]]     # рядки 0-2 включно (!), дві колонки
people.loc[people["age"] > 30, "name"] # булева маска → імена всіх старших 30
```

Ключовий нюанс: у `.loc[0:2]` **обидві межі inclusive** — це не як Python slicing. Причина: `.loc` працює з мітками, а не позиціями. "Від мітки 0 до мітки 2" означає всі три.

#### В) `.iloc[start:stop]` — доступ по позиціях

```python
people.iloc[0, 1]        # рядок з позицією 0, колонка з позицією 1
people.iloc[:3]          # перші 3 рядки (exclusive, як Python slicing)
people.iloc[-1]          # останній рядок
```

`.iloc` — "NumPy-style": позиції, exclusive `[start:stop)`.

**Коли який:**
- `df["col"]` — швидко витягнути колонку/колонки.
- `.loc` — коли фільтруєш за умовою, працюєш з іменами колонок, або індекс не `0..n-1`.
- `.iloc` — коли треба "перший рядок", "останні три", без прив'язки до міток.

### 4. Boolean masks — фільтрація

pandas НЕ приймає SQL-style `WHERE age > 22`. Замість цього — **булева маска**:

```python
# Крок 1: маска (Series з True/False довжини таблиці)
mask = people["age"] > 22
# 0     True
# 1     True
# 2    False
# 3    False
# 4    False

# Крок 2: застосувати маску до DataFrame
filtered = people[mask]
#    name  age  salary   city
# 0  Ігор   25  2000.0   Київ
# 1  Петро  23  1800.0  Львів

# Крок 3: аgregation
avg = filtered["salary"].mean()  # → 1900.0
```

**Ключова ідея:** `people["age"] > 22` — це не одне порівняння. Це **5 порівнянь одночасно** (по одному на рядок), результат — Series з True/False. Це broadcasting з NumPy, той самий принцип, тільки над іменованою колонкою.

**Gotcha:** для комбінації умов — `&`, `|`, `~` (не `and`, `or`, `not`), і кожна умова в дужках:
```python
people[(people["age"] > 22) & (people["city"] == "Київ")]
```

### 5. `groupby` — агрегація по групах

Ментально: `df.groupby("col")` — це "розкидай рядки по кучках, де в кожній кучці однакове значення `col`". Далі кажеш що робити з кожною кучкою.

**Три патерни:**

#### Патерн 1 — розмір кучок

```python
people.groupby("city").size()
# city
# Київ     2
# Львів    2
# Одеса    1
```

`.size()` рахує **рядки в кучці**, включно з NaN. Відрізняється від `.count()`, який рахує тільки non-null у конкретній колонці.

#### Патерн 2 — агрегація однієї колонки

```python
people.groupby("city")["salary"].mean()
# city
# Київ     1950.0
# Львів    1650.0
# Одеса    1800.0
```

Читай справа наліво: "**середнє** по колонці **salary** для кожного значення **city**".

Синтаксис — ланцюг:
1. `.groupby(<по чому>)` → об'єкт-група.
2. `[<яка колонка>]` → фокусуємось.
3. `.mean()` / `.sum()` / `.min()` / `.max()` → агрегація.

#### Патерн 3 — різні агрегації на різних колонках

```python
people.groupby("city").agg({
    "salary": "mean",           # одна операція
    "age": ["min", "max"],      # список операцій
})
```

Результат — DataFrame з multi-level колонками:
```
        salary    age
          mean    min  max
city
Київ    1950.0    22   25
Львів   1650.0    20   23
Одеса   1800.0    22   22
```

**Хороша практика — виносити groupby-об'єкт у змінну** і перевикористовувати:

```python
by_city = people.groupby("city")
by_city.size()
by_city["salary"].mean()
by_city.agg({"age": ["min", "max"]})
```

Це те що робиться в Jupyter — груповий об'єкт створюється раз, далі до нього чіпляють різні агрегації.

### 6. Реальний EDA — Titanic

```python
import seaborn as sns
titanic = sns.load_dataset("titanic")   # 891 рядок, 15 колонок
```

**Крок 1 — структура (`.head()`, `.info()`).**
Побачив дублюючі колонки: `survived` (0/1) і `alive` (yes/no) — одне і те саме. `pclass` (1/2/3) і `class` (First/Second/Third) — теж. При моделюванні не подавай обидві в фічі.

**Крок 2 — якість даних (`.isna().sum()`).**
```python
titanic.isna().sum()
# age            177     ← 20% пропусків
# deck           688     ← 77% (майже порожня)
# embarked         2
```

Це рішення перед моделюванням: `deck` — викидати чи заповнювати? `age` — імпутувати чи вирізати рядки?

**Крок 3 — базові метрики.**

Survival rate одним рядком через трюк "`.mean()` над бінарною колонкою":
```python
titanic["survived"].mean()   # 0.383... → 38%
```

Це працює бо `survived` містить `0` і `1`. Сума одиниць ділена на count = фракція одиниць = survival rate.

**Крок 4 — гіпотези через groupby.**

```python
titanic.groupby("sex")["survived"].mean()
# female    0.742    ← 74%
# male      0.189    ← 19%
```

Різниця в 4 рази. Стать — найсильніший single-feature signal.

```python
titanic.groupby("pclass")["survived"].mean()
# 1    0.630
# 2    0.473
# 3    0.242
```

Клас — сильний градієнт, але слабший за стать.

**Крок 5 — комбіновані фічі.**

```python
titanic.groupby(["sex", "pclass"])["survived"].mean()
# female  1    0.968
#         2    0.921
#         3    0.500
# male    1    0.369
#         2    0.157
#         3    0.135
```

Побачив що жінка в 1-му класі — 97% виживання, чоловік у 3-му — 13%. **Features не незалежні** — вони перемножуються. Це важлива інтуїція для feature engineering у Фазі 2.

### 7. EDA як методологія

Що дало це п'ять кроків EDA:

1. **Знайшли проблеми з якістю** (`deck` 77% NaN — треба рішення).
2. **Сформували гіпотези** (стать → survival — сильний сигнал; клас → survival — теж сильний, але слабший).
3. **Sanity-check припущень** (більшість пасажирів у 3-му класі, а не в 1-му).

**Ключове:** якщо не зробити EDA і одразу подати дані в модель — модель побачить статистику, але не побачить помилок джерела. `garbage in → garbage out`. EDA — це не додаток, це необхідний перший крок.

## Gotchas

- **`df["col"]` повертає Series, `df[["col"]]` повертає DataFrame з однією колонкою.** Не одне й те саме — форма і API різні.
- **`("name", "city")` — це tuple, `["name", "city"]` — list.** У `df[[...]]` потрібен list. `KeyError: ('name', 'city')` = tuple потрапив куди чекали список.
- **`.loc` — межі inclusive; `.iloc` — межі exclusive** (як Python slicing). Різна семантика бо різна природа доступу (label vs position).
- **`&`, `|`, `~` замість `and`, `or`, `not`** у boolean masks. І дужки навколо кожної умови.
- **`NaN` у pandas — перевантажений.** Може означати "справжнє missing" (пропуск у даних) або "не застосовується" (як у `describe(include="all")`). Треба звикнути читати контекст.
- **`.get("col")` vs `df["col"]`** — `.get()` тихо повертає `None` на опечатці, `[]` кидає `KeyError`. Використовуй `[]` — fail-fast.
- **`.size()` vs `.count()` у groupby** — `.size()` рахує всі рядки, `.count()` — тільки non-null у колонці. Різниця з'являється при NaN.
- **Забути `print()`** — код нічого не виведе, але не впаде. Класика "чому нічого не сталось". Завжди запусти і перевір вивід.
- **`import` посеред файлу** — PEP 8 вимагає всі імпорти на початку файлу. Ruff підсвітить (`E402`).
