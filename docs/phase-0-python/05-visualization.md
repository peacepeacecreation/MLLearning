# Урок 05 — matplotlib + seaborn: візуалізація Titanic

## Що це

**matplotlib** — базова бібліотека графіків у Python (з 2003 року). Написана за принципами MATLAB — звідси і specific ментальна модель:
- **Figure** — вся картинка (canvas).
- **Axes** — один графік всередині figure. У figure можуть бути кілька axes (subplots).

Ти пишеш низькорівнево: створи figure, додай axes, малюй, задавай labels/ticks/legend руками.

**seaborn** — надбудова над matplotlib. Спеціалізується на статистичних графіках, розуміє pandas DataFrame нативно, має красиві дефолти. Один API-контракт для всіх графіків:

```python
sns.xxx(data=df, x="col1", y="col2", hue="col3")
```

**Аналогія:**
- matplotlib = HTML/CSS з нуля.
- seaborn = React-компоненти зверху.

У реальному коді часто використовують **обидві разом**: seaborn для швидкого прототипу графіка, matplotlib — для дошліфовки (заголовки, кастомні кольори, легенди).

## Для чого

Візуалізація — це не "прикрашання" даних. Три конкретні речі які графіки роблять принципово краще ніж таблиці:

1. **Швидкість pattern-recognition.** Мозок обробляє 6×6 матрицю кольорів за 1-2 секунди, а 36 чисел — за 20-30. У 20×20 датасеті таблицю просто не читаєш очима, а heatmap скануєш. Це cognitive load argument.
2. **Форма розподілу.** З `describe()` ти бачиш `mean=29.7` для віку. Але яка форма? Нормальний дзвіночок? Двохгорбий? Skewed вправо? Це видно тільки з гістограми.
3. **Портфоліо.** У CV пишеш "робив EDA на Titanic". Без графіків — це рядок. З графіками — це реальний звіт, ready для показу.

Технічно: візуалізація — це **другий канал** передачі того самого сигналу з даних. Числа кажуть "female survival = 0.74", графік каже "female bar висока, male низька — стать домінує". Той самий факт, різна швидкість засвоєння.

## Розбір

### 1. matplotlib figure state — critical gotcha

matplotlib має **"поточний figure"** state. Кожен виклик `sns.xxx(...)` або `plt.xxx(...)` малює на **тому самому figure** — доки ти явно не викличеш `plt.show()`, `plt.figure()` або `plt.close()`.

**Що ламається без цього:**

```python
sns.histplot(data=titanic, x="age")
sns.countplot(data=titanic, x="pclass")
sns.boxplot(data=titanic, x="pclass", y="fare")
plt.show()   # ← одне на три виклики
```

Тут ти отримаєш **три графіки намальовані один поверх одного**: осі змішані, легенди дубльовані, значення на різних шкалах злиті в одну.

**Правильно:**

```python
sns.histplot(data=titanic, x="age")
plt.show()

sns.countplot(data=titanic, x="pclass")
plt.show()

sns.boxplot(data=titanic, x="pclass", y="fare")
plt.show()
```

Кожен `plt.show()` показує накопичене і **очищує** figure для наступного.

**Модерніший спосіб** — явно керувати fig/ax:

```python
fig, ax = plt.subplots()
sns.histplot(data=titanic, x="age", ax=ax)
plt.show()
```

`ax=ax` каже seaborn "малюй саме на цьому axes". Це чистіший стиль, і без нього ти не зможеш робити subplots (кілька графіків в одному вікні).

**У Jupyter це не проблема** — кожна cell створює новий figure автоматично. Але у `.py`-скрипті треба руками.

### 2. Три базові типи графіків

#### histplot — розподіл однієї числової змінної

```python
sns.histplot(data=titanic, x="age", bins=30)
plt.show()
```

- `bins=30` — кількість "кошиків" на які розбиваємо діапазон.
- Y-вісь = кількість пасажирів у кожному кошику.
- Побачиш форму: пік в 20-30, довгий хвіст праворуч (мало старших).

#### countplot — кількість рядків по категоріях

```python
sns.countplot(data=titanic, x="pclass")
plt.show()
```

- Y-вісь = **raw count**, без агрегацій.
- Це те саме що `titanic.groupby("pclass").size()` — просто картинкою.

#### boxplot — квартилі + outliers

```python
sns.boxplot(data=titanic, x="pclass", y="fare")
plt.show()
```

Boxplot читається так:
- **Коробка** = 25-75 перцентилі (IQR — interquartile range).
- **Лінія всередині** = медіана.
- **"Вуса" зверху/знизу** — розкид без outliers.
- **Точки поза вусами** = outliers (викиди).

На Titanic побачиш що 1-й клас має широчезний розкид fare з екстремальним outlier ~512$ (той хто заплатив за люкс-каюту).

### 3. Порівняння груп — barplot з `hue`

Це найпотужніший seaborn-паттерн. Комбінує три виміри в одному графіку:

```python
sns.barplot(data=titanic, x="pclass", y="survived", hue="sex")
plt.show()
```

- `x="pclass"` — категорія по горизонталі (три класи).
- `y="survived"` — числова колонка яка агрегується (default `.mean()`).
- `hue="sex"` — розбиває кожен pclass-стовпчик на підстовпчики по статі.

Результат: 6 стовпчиків (3 класи × 2 статі), висота = survival rate.

**Автоматика:** оскільки `survived` це 0/1, seaborn рахує `.mean()` і малює висоту як survival rate. Chorna riska на кожному стовпчику — confidence interval (bootstrap). Ти бачиш і значення, і його стабільність.

Це те саме що `titanic.groupby(["pclass", "sex"])["survived"].mean()` — тільки картинкою. Візуально одразу читаєш "жінка-1клас 97%, чоловік-3клас 13%".

### 4. Countplot vs Barplot — критична різниця

Спочатку легко плутати. Правило:

| | countplot | barplot |
|---|---|---|
| Що рахує | Кількість рядків у категорії | Агрегацію (mean за замовчуванням) |
| Y-вісь | Абсолютне число | Середнє y-колонки |
| Синтаксис | `x=<категорія>` | `x=<категорія>, y=<число>` |
| Приклад | "Скільки людей у 3-му класі" | "Яке середнє survival у 3-му класі" |

Для survival rate → barplot. Для "скільки людей у групі" → countplot.

### 5. Heatmap — матриця кореляцій

Це серйозніший інструмент — показує **всі числові фічі проти всіх** в одному графіку.

**Крок 1 — обчислити матрицю:**

```python
numeric = titanic.select_dtypes(include="number")
corr = numeric.corr()
```

- `select_dtypes(include="number")` — тільки числові колонки (не можна корелювати рядки).
- `.corr()` — pandas метод, повертає квадратну матрицю Pearson-коефіцієнтів між кожною парою колонок.

**Крок 2 — намалювати:**

```python
sns.heatmap(corr, annot=True, cmap="coolwarm", center=0, fmt=".2f")
plt.title("Кореляційна матриця Titanic")
plt.show()
```

Параметри:
- `annot=True` — виведе числа в клітинках.
- `cmap="coolwarm"` — палітра синій→білий→червоний.
- `center=0` — центрує палітру щоб білий = нуль.
- `fmt=".2f"` — 2 знаки після коми.

**Що читати на Titanic-матриці:**

- **Діагональ = 1.00 завжди** — кожна колонка ідеально корелює з собою (артефакт, не info).
- **Матриця симетрична** — `corr(A,B) = corr(B,A)`, тому верхній трикутник дублює нижній. У продакшн-графіках часто маскують один трикутник через `np.triu`.
- **`survived vs pclass = -0.34`** — помірна від'ємна. Вищий клас (менше число) → більше виживання.
- **`survived vs fare = +0.26`** — помірна додатна. Дорожчий квиток → виживання (через клас, див. нижче).
- **`pclass vs fare = -0.55`** — найсильніша тут. Клас і тариф — практично одне і те саме, тільки в різних форматах. Мультиколінеарність.
- **`survived vs age = -0.08`** — майже нуль. Вік сам по собі — слабкий signal (але це артефакт Pearson — див. пункт 7).

### 6. Pearson-кореляція під капотом

Формула:

```
r = cov(X, Y) / (std(X) × std(Y))
```

**Коваріація** (зверху) — це "чи ходять X і Y разом навколо своїх середніх":

```
cov(X, Y) = mean((X - mean_X) × (Y - mean_Y))
```

Для кожної точки береш **відхилення від середнього X** і **відхилення від середнього Y**, множиш. Потім усереднюєш.

Знак добутку:

| xi відносно mean_X | yi відносно mean_Y | добуток |
|---|---|---|
| вище | вище | + |
| нижче | нижче | + (два мінуси) |
| вище | нижче | − |
| нижче | вище | − |

Тобто **коваріація додатна коли X і Y рухаються в один такт** (обидва вгору або обидва вниз від своїх середніх).

**Мінімальний приклад** — 3 пасажири:

```
mean_fare = 30, mean_survived = 0.5

П1: fare=100, survived=1  → dev=(+70, +0.5) → +35
П2: fare=10,  survived=0  → dev=(-20, -0.5) → +10
П3: fare=200, survived=0  → dev=(+170, -0.5) → -85

sum = -40
cov = -40 / 3 = -13.3
```

**Ділення на `std × std`** — нормалізує одиниці. Без цього коваріація залежить від масштабу (фунти vs кілограми). З нормалізацією результат unit-less в `[-1, 1]` — можна порівнювати різні пари фіч.

**Геометрична інтуіція:** Pearson-корреляція = **косинус кута між mean-centered векторами**. `cos(0°) = 1` — ідеально прямо, `cos(90°) = 0` — перпендикулярно (незалежно), `cos(180°) = -1` — протилежно.

### 7. Обмеження Pearson — тільки лінійне

Класичний приклад:

```
X = [-2, -1, 0, 1, 2]
Y = X² = [4, 1, 0, 1, 4]
```

Візуально Y повністю залежить від X (парабола). Але:

```
mean_X = 0, mean_Y = 2

deviations:
   (-2, +2)  → -4
   (-1, -1)  → +1
   ( 0, -2)  → 0
   (+1, -1)  → -1
   (+2, +2)  → +4

sum = 0 → r = 0
```

Позитивні і негативні внески **точно скасовуються**. Pearson бачить "нема лінійного зв'язку" — і техно правий, зв'язок нелінійний.

Для monoтонних (але не лінійних) залежностей — **Spearman** (ранги замість значень). Для загальних — **mutual information** з теорії інформації.

**Практично для Titanic:** `age vs survived = -0.08` не означає "вік не важливий". Може бути що вік нелінійно впливає (діти виживають краще, дорослі — гірше, старші — так собі). Pearson усереднить це в майже нуль. Розділити зумієш через `sns.violinplot(x="survived", y="age", ...)` або кастомний binning.

### 8. Correlation ≠ Causation — confounder problem

Найпоширеніша пастка інтерпретації кореляцій. **Кореляція = "рухаються разом", каузація = "одне спричиняє інше".**

Класичний приклад — бар і пиво:

Ти знайшов `correlation(age, beer_consumption) = +0.7`. Спокуса сказати "старші п'ють більше пива". Але:

```
Реальний механізм:
   вік → частота відвідування бару → бар подає пиво → пиво

Тобто bar_visits — це confounder (прихований змішувач).
```

Якби ти виміряв bar_visits як окрему фічу:
- `correlation(bar_visits, beer) = +0.95` — дуже сильна
- `correlation(age, bar_visits) = +0.5` — помірна
- `correlation(age, beer | control bar_visits) → ~0` — падає до нуля

Тобто якщо порівнювати людей однакової частоти візитів у бар — вік перестає предиктувати пиво.

**Той самий патерн на Titanic:**

```
fare → pclass → каюти нагорі → шлюпки поруч → виживання
```

`fare` корелює з `survived` (+0.26) не тому що ціна причина, а тому що ціна є **проксі для класу**. Справжній механізм — фізична геометрія корабля (розташування кают і шлюпок).

**Практично для ML:**
- Кореляція — швидкий скринінг фіч, не остаточна відповідь.
- Модель може використовувати fare для предикту — це працює.
- Але не сказати "змусимо всіх купити дорогі квитки → більше виживуть". Це вимагає **інтервенції**, а не спостереження.

## Gotchas

- **`plt.show()` після кожного графіка у `.py` файлі** — інакше все накладається. У Jupyter це не потрібно.
- **countplot без `y=`, barplot з `y=`.** Плутають регулярно. Якщо `y=` це числова колонка → barplot агрегує (default mean). Якщо `y=` немає → countplot рахує.
- **`hue` як третій вимір без 3D-графіка.** Треба звикнути — це найпотужніший seaborn-паттерн.
- **Діагональ heatmap = 1.00 завжди.** Це артефакт, не інформація. Матриця симетрична — верхній трикутник дублює нижній.
- **Pearson бачить тільки лінійне.** `r ≈ 0` не завжди означає "немає зв'язку" — може бути нелінійний. Перевіряй через scatter plot.
- **Correlation ≠ causation.** `r > 0` не означає "X впливає на Y". Може бути confounder. Кузальний висновок вимагає експериментів або causal inference методів.
- **`include="number"`, не `includes`.** Класична опечатка з `s`. pandas сам підкаже "Did you mean 'include'?" — читай останній рядок traceback.
- **`corr()` тільки на числових колонках.** Тому спочатку `select_dtypes(include="number")`.
- **`|r| > 0.7` між фічами** — червоний прапорець мультиколінеарності. Може треба одну викинути.
