# Градієнтний спуск з нуля на NumPy

## Що це

Gradient descent — алгоритм мінімізації функції (loss) через ітеративне зсування параметрів у напрямку `-градієнт`. Цикл з 4 кроків:

```
1. Обчислити прогноз (forward)
2. Обчислити loss
3. Обчислити градієнт loss за параметрами
4. w = w - lr * grad   (оновлення в напрямку -градієнт)
```

Повторюється N разів, поки loss не стабілізується.

## Для чого

Це буквально те, як навчається будь-яка нейромережа, включно з великими мовними моделями. Різниця лише в масштабі: замість 1-2 параметрів (`w`, `b`) — мільярди, замість простого chain rule — backpropagation (той самий chain rule, застосований багато разів через шари).

## Розбір

### Мінімальний приклад

```python
w = 0.0
lr = 0.1
for step in range(50):
    grad = 2 * (w - 3)     # ∂L/∂w для L(w)=(w-3)²
    w = w - lr * grad
```

`w` збігається до `3` (мінімум), loss до `0`.

### Learning rate — головна ручка

- `lr` замалий (`0.01`) → loss повзе дуже повільно
- `lr` в самий раз (`0.1`) → плавно збігається
- `lr` завеликий (`1.0`) → осцилює **назавжди**, не збігаючись (loss застряг на одному значенні)
- `lr` набагато завеликий (`1.5`) → розходиться експоненційно до нескінченності

**Чому:** для `L(w)=(w-3)²`, кожен крок множить помилку `(w-3)` на коефіцієнт `(1 - 2*lr)`. Якщо `|1-2lr| < 1` — збіжність. Якщо `=1` — вічна осциляція. Якщо `>1` — розбіжність. Конкретне граничне значення `lr` залежить від "крутизни" (другої похідної) кожної функції — для інших loss воно буде іншим.

**Практична евристика:** починай з малого `lr` (0.01 чи 0.001). Якщо loss зростає/стрибає — зменш у 10 разів. Якщо повзе занадто повільно — збільш у 10 разів.

### Лінійна регресія з нуля

Модель `ŷ = w*x + b`, loss MSE `(1/N)*Σ(y_i-ŷ_i)²`. Градієнти (виведені в Уроці 04):

```python
y_pred = w * X + b
error = y_true - y_pred
grad_w = -2 * np.mean(X * error)
grad_b = -2 * np.mean(error)
w -= lr * grad_w
b -= lr * grad_b
```

Loss стабілізується на рівні дисперсії шуму в даних (наприклад, якщо шум `0.5*np.random.randn`, фінальний loss ≈ `0.5²=0.25`) — це теоретична межа (irreducible error), краще GD не зробить, і не повинен.

### Multiple linear regression

`X` shape `(N, d)`, `w` вектор `(d,)`, `ŷ = X @ w + b`:

```python
grad_w = -2 * (X.T @ error) / len(error)   # shape (d,) — той самий shape що w
grad_b = -2 * np.mean(error)
```

**Gotcha:** `X.T @ error` вже дає правильну форму `(d,)` (один градієнт на кожну фічу). Обгортання в `np.mean(...)` навколо цього вектора **зводить усі фічі до одного середнього числа** — усі ваги `w` тоді оновлюються однаково і "зливаються" в одне значення. Симптом бага: всі елементи `w` стають однаковими між собою, потім часто вибухає до `inf`.

### Batch / SGD / mini-batch (тільки термінологія)

- **Batch GD** — весь датасет на кожному кроці (те, що ми написали). Точний градієнт, повільний на великих даних.
- **SGD** — одна випадкова точка за крок. Шумний, швидкий.
- **Mini-batch** — 32/64/128 точок за крок. Індустрійний стандарт (у PyTorch — `DataLoader(batch_size=32)`).

### Нормалізація features

`X_norm = (X - X.mean(axis=0)) / X.std(axis=0)` — вирівнює масштаб фіч перед GD. Якщо одна фіча має розкид `0.001`, а інша `1000` — loss surface стає сильно витягнутим "яром", і GD рикошетить між стінками замість прямого руху до мінімуму. Після нормалізації surface стає більш "круглим", GD іде прямо.

## Gotchas

- **Стабільна осциляція ≠ збіжність.** Loss може "застрягти" на постійному значенні (не нуль) — це ознака завеликого `lr`, не успіху.
- **Знак:** `w -= lr*grad`, не `+=` — градієнт вказує напрямок **зростання**, рухаємось у протилежний бік.
- **`np.mean()` навколо вектора-градієнта** — легко випадково звести вектор `(d,)` до скаляра, зламавши multiple regression.
- **Забути reset параметрів між експериментами** — якщо тестуєш різні `lr` в циклі, обов'язково почни з `w=0` (чи іншого фіксованого старту) щоразу, інакше порівняння нечесне.
- **Без `np.random.seed()`** кожен запуск генерує нові випадкові дані — коректне порівняння швидкості збіжності між запусками вимагає фіксованого seed.
