# Похідні, часткові похідні, градієнт, chain rule

## Що це

Похідна `f'(x)` показує, наскільки швидко змінюється `f`, коли `x` трохи зростає. Часткова похідна — те саме, але для функції з кількома змінними, де всі інші змінні "заморожені" як константи. Градієнт — вектор часткових похідних за всіма параметрами одразу, і він вказує напрямок найшвидшого **зростання** функції.

## Для чого

Це основа оптимізації параметрів моделі. Маєш `loss = 5.3` і тисячі параметрів — градієнт каже, у який бік зсунути кожен параметр, щоб loss впав. `-градієнт` — напрямок найшвидшого спадання, тобто це і є один крок навчання (gradient descent, Урок 05).

## Розбір

### Таблиця похідних (запам'ятати, не виводити)

| f(x) | f'(x) |
|---|---|
| c (константа) | 0 |
| x | 1 |
| x² | 2x |
| x^n | n·x^(n-1) |
| e^x | e^x |
| ln(x) | 1/x |

Правила: `(f+g)' = f'+g'`, `(c·f)' = c·f'`, product rule `(f·g)' = f'g + fg'`.

### Знак похідної каже куди рухати параметр

```
L(w) = (w - 3)²,  L'(w) = 2(w-3)

L'(5) = 4   (додатне)  → якщо збільшити w, L зросте → зменшити w
L'(1) = -4  (від'ємне) → збільшити w
L'(3) = 0   → мінімум
```

Це і є ідея gradient descent у мініатюрі.

### Часткові похідні

Для `f(x,y) = x²y + xy² + 5`, рахуючи `∂f/∂x` — `y` вважається константою:

```
∂f/∂x = 2xy + y²    (доданок xy² = y²·x, похідна по x дає коефіцієнт y²)
∂f/∂y = x² + 2xy     (симетрично, заморозили x)
```

### Градієнт

```
∇f = [∂f/∂x1, ∂f/∂x2, ..., ∂f/∂xn]
```

Аналогія: туманний схил гори. Обмацуєш ногою куди стрімкіше вгору — це градієнт. Крок у протилежний бік — крок gradient descent.

### Chain rule (основа backpropagation)

Для композиції `y = f(g(x))`: `dy/dx = f'(g(x)) * g'(x)` — похідна зовнішньої (у точці внутрішньої) помножена на похідну внутрішньої.

```
y = e^(x²)
u = x²           →  du/dx = 2x
y = e^u          →  dy/du = e^u = e^(x²)
dy/dx = e^(x²) * 2x
```

**Найпоширеніша помилка:** забути домножити на похідну внутрішньої частини. `(3x+1)^5` → правильно `15*(3x+1)^4` (5 із степеня, помножене на 3 з `du/dx`), а не просто `5*(3x+1)^4`.

### Виведення градієнта MSE loss (це і є основа Уроку 05)

Модель `ŷ = w*x + b`, loss `L = (y - ŷ)²`. Розкладаємо: `L = e²`, де `e = y - ŷ`.

```
∂L/∂e = 2e
∂e/∂ŷ = -1        (мінус — бо ŷ входить зі знаком мінус у e = y - ŷ)
∂ŷ/∂w = x
∂ŷ/∂b = 1

∂L/∂w = ∂L/∂e * ∂e/∂ŷ * ∂ŷ/∂w = 2e * (-1) * x = -2x(y - ŷ)
∂L/∂b = ∂L/∂e * ∂e/∂ŷ * ∂ŷ/∂b = 2e * (-1) * 1 = -2(y - ŷ)
```

Це буде `grad_w` і `grad_b` у наступному уроці.

### Numerical gradient (перевірка)

```python
def numerical_gradient(f, params, h=1e-5):
    result = {}
    for key in params:
        params_plus = params.copy()
        params_plus[key] = params[key] + h
        params_minus = params.copy()
        params_minus[key] = params[key] - h
        result[key] = (f(**params_plus) - f(**params_minus)) / (2 * h)
    return result
```

Апроксимує похідну через `(f(x+h) - f(x-h)) / (2h)`. У проді не використовується (повільно — два виклики моделі на кожен параметр), але для **gradient check** (перевірки аналітичного градієнта) — стандартна техніка.

## Gotchas

- **Похідна — це нова функція, не рівняння для розв'язання.** Легко сплутати "візьми похідну" з "знайди x".
- **Забути домножити на `du/dx` в chain rule** — найчастіша помилка (`(3x+1)^5` без множника `3`).
- **Sign error:** `∂(y - ŷ)/∂ŷ = -1`, не `+1` — мінус перед `ŷ` легко загубити.
- **Підстановка символів у формулу (product rule, chain rule)** — знати формулу і правильно підставити в неї конкретні `f, g, f', g'` — різні навички. Підставляти по одному шматочку, не всі одразу.
- **`d/dx` — це просто нотація**, альтернативний запис до `f'(x)`, не нова концепція.
