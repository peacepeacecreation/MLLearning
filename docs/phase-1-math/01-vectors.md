# Вектори, скалярний добуток, cosine similarity

## Що це

Вектор — це список чисел, який можна уявити як стрілку з напрямком (не тільки як набір цифр). Dot product (скалярний добуток) і cosine similarity — дві операції над парою векторів, які міряють кут між цими стрілками: чим менший кут, тим більша схожість.

## Для чого

Це та сама математика, яку використовує RAG-пошук (наприклад, коли ChatGPT шукає релевантний фрагмент у базі знань): текст перетворюється на embedding-вектор, і "наскільки схожі два тексти" рахується саме через cosine similarity між їхніми embedding-ами. Та сама операція лежить в основі attention-механізму трансформерів (порівняння query і key векторів) і рекомендаційних систем.

## Розбір

### Вектор і базові операції

Вектор — `np.array([1, 2, 3])`. Ключовий gotcha: у звичайному Python-списку `+` — це конкатенація, у NumPy — поелементне додавання:

```python
[1, 2, 3] + [4, 5, 6]              # [1, 2, 3, 4, 5, 6] — конкатенація
np.array([1,2,3]) + np.array([4,5,6])   # [5, 7, 9] — поелементно
```

### Норма (довжина вектора)

```python
np.linalg.norm(v)   # sqrt(v[0]**2 + v[1]**2 + ...) — Піфагор у n вимірах
```

### Dot product (скалярний добуток)

Два еквівалентні визначення:
- **Алгебраїчне:** сума поелементних добутків — `a[0]*b[0] + a[1]*b[1] + ...`
- **Геометричне:** `a · b = ||a|| * ||b|| * cos(θ)`, де θ — кут між векторами

```python
a @ b            # або np.dot(a, b)
```

Знак результату каже про напрямок: `> 0` — гострий кут (схожий напрямок), `= 0` — перпендикулярно, `< 0` — протилежний напрямок.

**Важливо:** `dot(a,b) = 0` не означає, що в масивах є нулі. Приклад: `[1,1] @ [1,-1] = 1*1 + 1*(-1) = 0`, хоча жоден елемент не нульовий — просто вектори перпендикулярні (90°).

### Cosine similarity

Проблема dot product: він росте з довжиною векторів, навіть якщо кут (реальна "схожість") той самий. Рішення — поділити на добуток норм:

```python
def cosine_sim(a, b):
    return (a @ b) / (np.linalg.norm(a) * np.linalg.norm(b))
```

Результат завжди в діапазоні `[-1, 1]`: `1` — однаковий напрямок, `0` — не пов'язані (перпендикулярні), `-1` — протилежний напрямок.

### Приклад: пошук найсхожішого кандидата (міні-RAG)

```python
def most_similar(query, candidates):
    return int(np.argmax([cosine_sim(query, c) for c in candidates]))
```

Цикл по кандидатах + пошук індексу з максимальною cosine similarity — це і є, у спрощеному вигляді, retrieval-крок у RAG: знайти документ, найбільш семантично близький до запиту.

## Gotchas

- **`list + list` (конкатенація) vs `np.array + np.array` (поелементно)** — різна поведінка того самого оператора, мовчазний баг якщо переплутати.
- **`a * b` vs `a @ b`** — `*` це поелементне множення (Hadamard product), `@` це скалярний добуток (одне число). Легко переплутати.
- **`dot(a,b) = 0` ≠ "нулі в масиві".** Це про геометрію (перпендикулярність), не про самі значення координат.
- **Cosine similarity не про "близькість чисел".** Два вектори можуть мати зовсім різні за величиною координати (`[100,100]` і `[1,1]`) і при цьому мати cosine similarity `= 1`, бо напрямок однаковий. І навпаки — числа можуть виглядати "схожими" за величиною, а напрямок (і отже схожість) — ні.
- **Нульовий вектор** робить cosine similarity невизначеним (ділення на 0 в нормі) — на практиці варто перевіряти окремо.
