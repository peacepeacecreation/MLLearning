# Лінійна регресія через sklearn

## Що це

`sklearn.linear_model.LinearRegression` — та сама лінійна регресія `ŷ = w*x + b`, яку ми писали вручну на NumPy у Фазі 1, тільки тепер тренується за 3 рядки коду замість власноручного циклу gradient descent.

## Для чого

Практичне застосування регресії на реальних даних (California housing — передбачення медіанної ціни житла з 8 ознак), і розуміння того, що saме відбувається "під капотом" `.fit()` — бо це не gradient descent, а зовсім інший метод.

## Розбір

### LinearRegression — це не gradient descent

Для звичайної (нерегуляризованої) лінійної регресії з MSE loss існує **аналітичний розв'язок** (closed-form), а не ітеративний пошук:

```
w = (XᵀX)⁻¹ Xᵀy
```

Це буквально **алгебраїчний розв'язок** рівняння `grad_w = 0` (тієї самої формули з Фази 1, `grad_w = -2*(X.T@error)/N`, прирівняної до нуля і розв'язаної відносно `w`). GD **шукає** цю точку ітеративно, малими кроками; normal equation **обчислює** її напряму, за один прохід. sklearn рахує це через SVD (чисельно стабільніше), суть та сама.

```python
model = LinearRegression()
model.fit(X_train, y_train)   # єдина дія: рахує w, b через формулу вище
```

**Важливо не плутати:** `.coef_` (результат) — це сам знайдений параметр `w`, **не** градієнт. Градієнт — лише умова (`=0`), яку алгебраїчно розв'язали, щоб отримати `w`. `grad_w=0` не означає `w=0` — приклад: `L(w)=(w-3)²`, `L'(w)=2(w-3)=0` дає `w=3`, а не `w=0`.

### .coef_ / .intercept_ = w / b

```python
model.coef_        # масив (d,), по одному числу на кожну ознаку — той самий w з Фази 1
model.intercept_   # скаляр — той самий b
```

Прогноз рахується тією самою формулою: `ŷ = X @ model.coef_ + model.intercept_`. `.coef_` прив'язаний до **порядку колонок** при `fit` — переставити колонки між `fit` і читанням `.coef_` = мовчазна помилка інтерпретації.

### R² — не accuracy

```
R² = 1 - (SS_residual / SS_total)
```

`R²=0.58` означає: модель пояснює **58% розкиду (варіативності)** цін на житло навколо середнього значення — тобто ознаки (дохід, розташування тощо) реально допомагають передбачати ціну краще за "завжди кажи середнє", але не ідеально. Це **не** "58% прогнозів правильні" — кожен прогноз має якусь помилку, R² каже про сукупну пояснену варіативність, не про точність окремих передбачень.

**MSE vs R²:** MSE — у квадратних одиницях `y` (тут — сотні тисяч доларів у квадраті), залежить від масштабу задачі, важко порівнювати між різними задачами. R² — безрозмірний, приблизно `[0,1]` (може бути від'ємним для дуже поганих моделей), порівнюваний між задачами незалежно від масштабу `y`.

### Чи потрібен StandardScaler для LinearRegression?

**Для збіжності — ні.** Closed-form розв'язок не ітерує і не залежить від масштабу даних, щоб знайти правильну відповідь. R² з `StandardScaler` і без нього — **ідентичний**.

**Але scaling усе одно потрібен для:**
1. **Інтерпретації `.coef_`** — без scaling ваги різних ознак (дохід у діапазоні 0-15 vs population у тисячах) непорівнювані між собою за "величиною впливу".
2. **Регуляризованих моделей (Ridge/Lasso, Урок 08)** — там штраф на вагу однаковий для всіх ознак, тому масштаб має значення для справедливості штрафу.

**Чому `.coef_` різні, а R² однаковий:** модель компенсує зміну масштабу `x`, пропорційно змінюючи `w`, так щоб добуток `w*x` (і відповідно прогноз) лишався тим самим. Тобто scaling не змінює **саму залежність** між `X` і `y` — тільки одиниці, у яких ця залежність виражена.

### Multicollinearity

Коли дві ознаки сильно корелюють між собою (не з `y`, а між собою — наприклад `Latitude`/`Longitude`, r=-0.92, чи `AveRooms`/`AveBedrms`, r=0.84), модель не може впевнено розділити вплив між ними — `.coef_` для цих ознак стають **нестабільними** (чутливими до шуму в даних), хоча сумарний прогноз (і `R²`) при цьому лишається нормальним. Діагностика — кореляційна матриця (`X_train.corr()`) чи VIF. Рішення (не реалізовано тут) — прибрати одну з ознак чи застосувати регуляризацію (Ridge).

## Gotchas

- **`.coef_` — не градієнт.** Це сам параметр (знайдена відповідь), градієнт — лише умова, з якої цю відповідь алгебраїчно вивели.
- **`R² ≠ accuracy.`** R²=0.6 не означає "60% прогнозів правильні" — це "модель пояснює 60% розкиду `y`".
- **`StandardScaler` ≠ `MinMaxScaler`.** Z-score не звужує значення в `[0,1]` — типовий діапазон приблизно `[-3,+3]` (32% значень мають `|z|>1` для нормального розподілу). У `[0,1]` звужує саме `MinMaxScaler`.
- **`.coef_` величина не обмежена жодним діапазоном** — залежить від сили впливу конкретної ознаки на `y` в конкретному датасеті, не від якогось універсального правила.
- **Multicollinearity псує `.coef_`, не обов'язково R².** Не плутати нестабільність ваг з поганою якістю прогнозу.
