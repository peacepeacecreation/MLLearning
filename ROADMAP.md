# 🗺️ ROADMAP — Мій шлях до ML-інженера

> **Профіль:** впевнений у JavaScript/TypeScript, базовий Python (швидко підтягну), мета — **робота ML-інженером**, темп **10–20 год/тиждень**.
> **Орієнтовна тривалість:** ~9–12 місяців до рівня junior/middle ML Engineer при стабільному темпі.

Легенда статусів: `[ ]` не почато · `[~]` в процесі · `[x]` завершено

---

## 🎯 Загальна логіка шляху

ML-інженер ≠ дата-саєнтист. Крім моделей, потрібні **інженерні навички**: чистий код, тестування, деплой, робота з даними в проді, MLOps. Твій досвід у TS/JS — велика перевага саме тут: інженерна культура вже є, залишається наростити математику, Python-екосистему та ML-специфіку.

Структура: спочатку фундамент (Python + математика), далі класичний ML, потім deep learning, далі спеціалізація (LLM/GenAI дуже гарячі), і паралельно — портфоліо + MLOps для працевлаштування.

---

## Фаза 0 — Фундамент Python та інструменти (3–4 тижні)

Мета: писати Python так само впевнено, як TS, і освоїти базовий інструментарій.

- [ ] Python-синтаксис для тих, хто знає іншу мову: типи, comprehensions, генератори, декоратори, context managers
- [ ] Type hints у Python (`mypy`) — тобі як TS-розробнику зайде легко
- [ ] Віртуальні середовища: `venv` / `uv` / `conda`, керування залежностями
- [ ] Jupyter / VS Code notebooks — робочий процес експериментів
- [ ] `git` для ML-проєктів (гілки, `.gitignore` для даних/моделей)
- [ ] Основи `NumPy` (масиви, broadcasting, векторизація)
- [ ] Основи `pandas` (DataFrame, фільтри, groupby, merge)
- [ ] Візуалізація: `matplotlib` + `seaborn`

**Ресурси:** офіційний Python tutorial, "Python for Data Analysis" (Wes McKinney), NumPy/pandas docs.
**Практика:** взяти CSV-датасет (напр. Titanic) і зробити повний EDA-ноутбук.

---

## Фаза 1 — Математика для ML (4–6 тижнів, паралельно з Фазою 2)

Мета: не вивести з нуля всю математику, а розуміти, що відбувається "під капотом" і читати формули без страху.

- [ ] Лінійна алгебра: вектори, матриці, множення, власні значення (3Blue1Brown "Essence of Linear Algebra")
- [ ] Похідні, градієнти, chain rule (основа backpropagation)
- [ ] Основи ймовірностей та статистики: розподіли, matematичне сподівання, дисперсія, Байєс
- [ ] Градієнтний спуск — інтуїція та реалізація вручну на NumPy
- [ ] Функції втрат (MSE, cross-entropy) — що вони означають

**Ресурси:** 3Blue1Brown (YouTube), "Mathematics for Machine Learning" (Deisenroth, безкоштовна PDF), Khan Academy для прогалин.
**Порада:** не застрягай надовго. Достатньо "робочого" розуміння — заглиблюйся за потребою.

---

## Фаза 2 — Класичний Machine Learning (6–8 тижнів)

Мета: розуміти й застосовувати основні алгоритми, знати повний ML-пайплайн.

- [ ] Supervised vs unsupervised, train/validation/test split
- [ ] Лінійна та логістична регресія
- [ ] Дерева рішень, Random Forest, Gradient Boosting (XGBoost / LightGBM)
- [ ] KNN, SVM, наївний Байєс
- [ ] Кластеризація: K-Means, ієрархічна
- [ ] Зниження розмірності: PCA
- [ ] Метрики: accuracy, precision/recall, F1, ROC-AUC, confusion matrix
- [ ] Перенавчання, регуляризація, крос-валідація
- [ ] Feature engineering та препроцесинг (`scikit-learn` pipelines)
- [ ] Підбір гіперпараметрів (GridSearch, Optuna)

**Ресурси:** Andrew Ng "Machine Learning Specialization" (Coursera), "Hands-On Machine Learning" (Géron) — ключова книга, `scikit-learn` docs.
**🏆 Проєкт 1:** end-to-end класифікація/регресія на реальному датасеті з Kaggle + звіт.

---

## Фаза 3 — Deep Learning (8–10 тижнів)

Мета: розуміти нейромережі й впевнено працювати з фреймворком.

- [ ] Перцептрон, багатошарові мережі, активації, backpropagation
- [ ] **PyTorch** (індустрійний стандарт) — тензори, autograd, `nn.Module`, тренувальний цикл
- [ ] Оптимізатори (SGD, Adam), learning rate scheduling, batching
- [ ] Регуляризація в DL: dropout, batch norm, early stopping
- [ ] CNN — комп'ютерний зір (класифікація зображень)
- [ ] RNN/LSTM — основи послідовностей (коротко, як історія)
- [ ] **Transformers** та attention — фундамент сучасного ML (обов'язково!)
- [ ] Transfer learning та fine-tuning готових моделей
- [ ] Робота з GPU (Google Colab / Kaggle GPU безкоштовно)

**Ресурси:** "Deep Learning Specialization" (Andrew Ng), fast.ai курс, PyTorch tutorials, "The Illustrated Transformer" (Jay Alammar).
**🏆 Проєкт 2:** класифікатор зображень на CNN АБО fine-tuning трансформера на своєму датасеті.

---

## Фаза 4 — Спеціалізація: LLM та GenAI (6–8 тижнів)

Мета: найгарячіша частина ринку. Тут твій full-stack/TS-бекграунд особливо цінний — багато вакансій "AI Engineer" про інтеграцію LLM у продукти.

- [ ] Як працюють LLM: токенізація, embeddings, контекстне вікно
- [ ] Prompt engineering системно
- [ ] Робота з API (OpenAI, Anthropic) — тут TS/JS теж у грі
- [ ] RAG (Retrieval-Augmented Generation): векторні БД (Chroma, pgvector, Pinecone)
- [ ] Фреймворки: LangChain / LlamaIndex
- [ ] Fine-tuning та LoRA/QLoRA (ефективне донавчання)
- [ ] Оцінка якості LLM-застосунків (evals)
- [ ] Агенти та tool use

**Ресурси:** Hugging Face курс (NLP + LLM), Anthropic/OpenAI docs, "Building LLM apps" туторіали.
**🏆 Проєкт 3:** RAG-застосунок (чат по власних документах) з фронтендом на React/TS — покаже і ML, і інженерію.

---

## Фаза 5 — MLOps та продакшн (4–6 тижнів, паралелиться з портфоліо)

Мета: те, що відрізняє ML-інженера від дата-саєнтиста. Твоя інженерна сила.

- [ ] Serving моделей: FastAPI, REST/gRPC ендпоінти
- [ ] Docker для ML-сервісів
- [ ] Експеримент-трекінг: MLflow / Weights & Biases
- [ ] Версіонування даних та моделей (DVC)
- [ ] CI/CD для ML-пайплайнів
- [ ] Моніторинг у проді: data drift, деградація моделі
- [ ] Хмара: базово AWS SageMaker АБО GCP Vertex AI
- [ ] Основи масштабування та інференс-оптимізації

**Ресурси:** "Designing Machine Learning Systems" (Chip Huyen) — маст-рід, "Made With ML" (madewithml.com, безкоштовно).
**🏆 Проєкт 4:** задеплой один зі своїх проєктів як живий сервіс з Docker + API + моніторинг.

## ✅ Принципи, щоб не вигоріти й не забуксувати

- **80% практики, 20% теорії.** Кодь одразу, теорію добирай за потребою.
- **Один проєкт важливіший за десять туторіалів.** Роботодавці дивляться на проєкти.
- **Не женися за ідеальним розумінням математики** — достатньо робочого.
- **Твоя перевага — інженерія.** Багато дата-саєнтистів слабкі в проді; ти — навпаки. Роби на цьому акцент.
- **Веди нотатки.** Записуй, що вивчив — і закріплюєш, і матеріал для блогу.
- **Ретроспектива щомісяця:** що зроблено, що застрягло, коригуй план.
