# /create-lesson — розгортання фази в повноцінні уроки

Аргумент: **$ARGUMENTS** — номер фази з ROADMAP.md (0..6).

## Що робиш

1. **Прочитай контекст проекту** (обов'язково, послідовно, поки не всі файли не в голові):
   - `CLAUDE.md` — правила проекту (Ukrainian, mentor mode, 80/20).
   - `ROADMAP.md` — знайди секцію `## Фаза $ARGUMENTS — ...`. Витягни всі буллети з цієї фази.
   - `PROGRESS.md` — на якому кроці зупинились (щоб врахувати передумови).
   - `docs/phase-$ARGUMENTS-*/` — що вже є в офіційних уроках учня (щоб не дублювати).
   - `lessons/phase-1/README.md` — **це джерело правди по методиці і формату**. Читай уважно; будеш наслідувати той самий стандарт.
   - `lessons/phase-1/lesson-*.md` — 1-2 файли як приклади структури.

2. **Провід research best-practices подачі саме цієї теми** (перед написанням):
   - Використай `context7` / `ref` MCP-сервер, якщо треба свіжу документацію по інструментах фази (наприклад `scikit-learn`, `pytorch`, `langchain`).
   - Не тягни абстрактну педагогіку — застосовуй ту саму методику з `lessons/phase-1/README.md` (concrete-first, just-in-time math, TS-analogies, active recall, spaced repetition, chunking, dual coding, desirable difficulties, emotional hooks).
   - Специфіку фази врахуй: Phase 2 (класичний ML) — sklearn-first, багато метрик; Phase 3 (deep learning) — PyTorch, тензори, autograd; Phase 4 (LLM) — API + RAG + evals; Phase 5 (MLOps) — Docker/FastAPI/деплой; Phase 6 (портфоліо) — не технічна.

3. **Розбий фазу на 6-8 уроків + capstone**. Керуйся принципами:
   - **Chunking:** 1-2 концепти на урок, не 5. Урок = 1.5-3 години активної роботи.
   - **Interleaving:** попередній урок повинен готувати гачок для наступного.
   - **Прогресія:** від "hello world" до інтеграції.
   - **Capstone обов'язково:** один урок наприкінці, що зшиває всі теми в один робочий міні-проект.
   - Загальний обсяг фази — відповідно ROADMAP-оцінці (Phase 2: 6-8 тижнів → 8-10 уроків; Phase 3: 8-10 тижнів → 10-12 уроків; Phase 4: 6-8 тижнів → 8-10; Phase 5: 4-6 тижнів → 6-8; Phase 6 — інша структура, це не технічна фаза, пропонуй окремо).

4. **Створи папку `lessons/phase-$ARGUMENTS/`** і напиши файли:
   - `README.md` — індекс + професійна методика саме для цієї фази + профіль учня + таблиця уроків + anti-patterns + критерії "фаза закрита".
   - `lesson-01-*.md`, `lesson-02-*.md`, ... , `lesson-NN-capstone.md`.

5. **Кожен урок має ті самі 13 секцій**, що у `lessons/phase-1/lesson-01-vectors.md`:
   1. **Мета уроку** — 1 речення про learning outcome.
   2. **Передумови** — з яких попередніх уроків/фаз.
   3. **Емоційний гачок** — 1 речення, навіщо це в реальному ML.
   4. **Ключові концепти + як подавати** — 3-6 підсекцій із хвилинами, TS-аналогіями, кодом.
   5. **Демо, яке пишеш на очах** — короткий copy-pasteable snippet.
   6. **Задачі для учня** — 4-6 задач від drill до pushback. Кожна з чіткою постановкою.
   7. **Контрольні питання** — 6-10 усних питань без підказки.
   8. **Розгорнуті відповіді** — правильна версія кожного контрольного, для перевірки себе.
   9. **Типові помилки, які я очікую** — 4-6 пунктів, кожен з коротким планом реакції.
   10. **Критерії "готово, можна далі"** — checklist з 4-6 пунктів.
   11. **Що повторити через тиждень/місяць** — spaced repetition пункти.
   12. **Джерела** — 3-5 посилань (3B1B, StatQuest, конкретні docs, книги з ROADMAP).
   13. **Урок для docs/** — bullet-план того, що учень напише у `docs/phase-$ARGUMENTS-*/NN-*.md` після практики.

6. **Правила якості (обов'язкові):**
   - Українською.
   - TS/JS-аналогії де є природні (Мартин — досвідчений TS-розробник).
   - **Не пиши код за учня.** У задачах — специфікація, не рішення. Рішення — у "Розгорнутих відповідях" або окремому блоці для ментора.
   - Не тягни зайвих залежностей на майбутнє. Тільки те, що потрібно у фазі.
   - Кожен урок — самодостатній (майбутнє я+рік має зрозуміти без контексту).
   - Позначай gotchas явно (це найцінніше у шпаргалці ментора).
   - Обсяг уроку — ~200-400 рядків markdown.

7. **Оновити `PROGRESS.md`** — додати рядок:
   > `lessons/phase-$ARGUMENTS/` розгорнуто в N уроків, готово до проходження.

   **НЕ** ставити фазу як пройдену — уроки готові, але учень ще не пройшов практику.

## Формат і локації файлів

- Папка: `lessons/phase-$ARGUMENTS/`.
- Індекс: `README.md`.
- Уроки: `lesson-NN-<topic-slug>.md`, `NN` — 2-значне, `<topic-slug>` — короткий kebab-case.
- Capstone: останній файл, назва `lesson-NN-capstone.md`.

## Правила поведінки

- Якщо папка `lessons/phase-$ARGUMENTS/` **вже існує і не пуста** — не перезаписуй мовчки. Скажи користувачу, що вже там, і спитай: overwrite / merge / abort.
- Якщо фаза $ARGUMENTS не існує в ROADMAP.md — так і скажи, не вигадуй.
- Якщо `$ARGUMENTS` порожній або нецифра — попроси вказати номер фази.
- Використовуй `TaskCreate` / `TaskUpdate` для трекінгу прогресу (по одному таску на файл), якщо буде більше 3 файлів (а їх буде більше).

## Anti-patterns (чого не роби)

- Не пиши уроки "з голови", ігноруючи `lessons/phase-1/README.md`. Це — стандарт формату; наслідуй його.
- Не роби короткі уроки типу "визначення + 1 задача". Це шпаргалка ментора, а не learning card — там мають бути 6+ задач, розгорнуті відповіді, аналіз типових помилок.
- Не пиши абстрактно ("це важлива тема, зрозуміти концепції"). Пиши конкретно ("він плутатиме `A * B` і `A @ B` — готую цей приклад заздалегідь").
- Не забувай про spaced repetition — кожен урок явно вказує, що повторювати з попередніх.
- Не дублюй те, що вже є в `docs/phase-$ARGUMENTS-*/`. Ці уроки — приватна підготовка ментора; вони готуються **до** практики учня, а офіційний docs-конспект пише сам учень **після**.

## Після завершення — відрапортуй

- Скільки уроків створено, разом рядків.
- Топ-3 інсайти, які були в фазі найважчі для подачі (з методичного погляду).
- Пропозиції по 1-му уроку — що учень має зробити першим на практиці.
