# /lesson-start

Стартує сесію нового уроку в Claude Mentor. Двоканальна доставка:

- **Термінал:** повний урок (теорія + приклади + задачі) за правилами `CLAUDE.md`.
- **Voice:** контекстний вступ до теорії + comprehension-check через `voice-claude`, спавниться паралельно.

## Крок 1 — аналіз стану

Прочитай паралельно:

- `memories/profile.md` — гранулярний рівень учня по підтемах (не "тема закрита", а per-subtopic).
- `PROGRESS.md` — закладка: остання закрита тема, черга наступних.
- `ROADMAP.md` — повний план, поточна фаза.
- Останній урок у `docs/phase-N-*/NN-topic.md` — на чому будуємо.

**Визначи наступну тему:**

- Якщо є `topic-hint` в аргументі → воно override-ить чергу.
- Інакше → наступна за `PROGRESS.md` + свіжа прогалина з `profile.md` (гранулярна тема з "плутається" / "не знав").

## Крок 2 — генерація ephemeral flow

Скомпонуй flow JSON, який точно відповідає стану **цього** учня. Запиши через `Write` у:

```
voice/flows/tmp/lesson-{topic-slug}.json
```

Формат:

```json
{
  "mode": "eli5-ua",
  "topic": "{topic-slug}",
  "difficulty": "easy",
  "duration_min": 5,
  "language": "uk",
  "interviewer_persona": "friendly_concept_introducer",
  "opening": "<2-3 речення чистої теорії українською, БЕЗ коду, побудовані на тому що учень демонстровано знає>",
  "questions": [
    {
      "id": "q1",
      "text": "<comprehension probe спрямований на прогалину що видно в profile.md>",
      "follow_ups": ["<глибше пояснення якщо учень плаває на найбільш ймовірному місці>"],
      "time_budget_sec": 60
    }
  ],
  "wrap_up": "Ок, готовий писати перший приклад?"
}
```

**Правила генерації opening/questions:**

- **Opening:** будується явно на тому що `profile.md` маркує як "впевнено" для сусідніх тем. Ніколи не пояснює те, що учень вже знає.
- **Questions:** цілять у концепт який `profile.md` показує як ймовірно шаткий (adjacent gap).
- **Мова:** українська, розмовна, без жаргону без миттєвого пояснення, **БЕЗ КОДУ**.
- **Тривалість opening:** 30-60 секунд говоріння загалом.

## Крок 3 — спавн voice-claude у background (термінальний режим)

Використай Bash з `run_in_background: true`:

```bash
cd voice && uv run python bot_cli.py --flow tmp/lesson-{topic-slug} > /tmp/voice-lesson-{topic-slug}.log 2>&1
```

**Чому саме `bot_cli.py`, не `launch.sh`:**
- `bot_cli.py` використовує `LocalAudioTransport` — голос грає через системний звук macOS. Ніякого браузера, учень лишається в терміналі.
- `launch.sh` розрахований на повноцінні інтервʼю з браузерним UI. Для intro-теорії коли учень паралельно читає текстовий урок — це перебивання контексту.

Транскрипт летить у `/tmp/voice-lesson-{topic-slug}.log` — можна `tail -f` під час сесії або читати після.

Якщо `bot_cli.py` падає (немає ключів, .env порожній) — крок 3 пропускається gracefully, текстовий урок доставляється як зазвичай.

## Крок 4 — паралельно писати повний урок у терміналі

Поки voice працює в background, доставляй повний урок за педагогікою `CLAUDE.md`:

- Пояснення концепту (текст може йти глибше ніж voice-opening — там компромісу немає)
- Приклади коду (voice свідомо їх не має — для тексту вони критичні)
- Специфікації задач за правилом сходинок підказок
- Джерела для поглиблення

**Не дублювати voice-opening слово-в-слово в тексті** — учень слухає або читає, покриваємо різні кути.

## Крок 5 — після завершення voice-сесії

- Транскрипт лежить у `/tmp/voice-lesson-{topic-slug}.log` (stdout з `bot_cli.py`)
- Ephemeral flow файл лишається в `voice/flows/tmp/` — очиститься на наступному `/end-lesson` або через 24 години
- Якщо учень скаже "перевір голосову" — читаю транскрипт (`/tmp/voice-lesson-*.log`) і оновлюю `memories/profile.md` за правилами Post-session review з `VOICE-TUTORIALS.md`

## Правило: НЕ комбінувати з `-s` маркером

`voice-claude` (bot_cli.py) використовує **Cartesia** через Pipecat pipeline — природний голос, sonic-3.

`TTS-piper` (той що активується `-s` маркером у промпті) — окрема система, локальний Piper, читає **мою текстову відповідь**. Він потрібен для приватності, але акустично помітно гірший.

**Якщо викликати `/lesson-start` з `-s` — грають ДВА голоси одночасно** (Cartesia inтро + Piper моя відповідь) = каша. Правильно: `/lesson-start` без `-s`. Voice-claude сам все скаже красиво через Cartesia, а я мовчки пишу текст у терміналі.

## Коли НЕ використовувати

- **Продовження попереднього уроку** з попередньої сесії — просто продовжуй розмову, `/lesson-start` тут зайвий.
- **Retrieval / повторення** пройденого — це інший flow (retrieval, не intro).
- **Debugging чи розв'язування задач** — це звичайний mentor-mode dialogue.

## Референс

Повна система голосового підкріплення — `VOICE-TUTORIALS.md`.
